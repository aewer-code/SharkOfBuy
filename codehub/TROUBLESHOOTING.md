# Решение проблем CodeHub

## Проблема: Backend возвращает 404

**Решение:**
1. Убедитесь, что backend запущен: `./start-backend.sh`
2. Проверьте подключение: `./check-backend.sh`
3. Backend должен быть на `http://localhost:5000`
4. Frontend должен быть на `http://localhost:8000`

## Проблема: SSL ошибка

**Решение:**
- Используйте `http://localhost:8000` (не https://)
- Или `http://127.0.0.1:8000`
- Очистите кеш браузера
- Используйте режим инкогнито

## Проблема: "Backend не запущен"

**Решение:**
1. Откройте новый терминал
2. Перейдите в папку проекта
3. Запустите: `./start-backend.sh`
4. Должно появиться: "Backend запущен на http://localhost:5000"

## Проблема: Посты не загружаются

**Решение:**
1. Проверьте консоль браузера (F12)
2. Убедитесь, что backend запущен
3. Проверьте, что используете правильный URL: `http://localhost:8000`
4. Проверьте CORS настройки (должны быть включены)

## Проверка работы:

```bash
# Проверить backend
curl http://localhost:5000/api/health

# Должен вернуть:
# {"status":"ok","message":"CodeHub API is running",...}
```

## Логи для отладки:

1. **Backend логи** - смотрите в терминале где запущен `./start-backend.sh`
2. **Frontend логи** - откройте консоль браузера (F12 → Console)
3. **Network запросы** - F12 → Network → проверьте запросы к `/api/*`

## Частые ошибки:

- `Failed to fetch` - backend не запущен или неправильный URL
- `404 Not Found` - неправильный endpoint или backend не запущен
- `CORS error` - проверьте настройки CORS в backend
- `SSL error` - используйте http:// вместо https://
